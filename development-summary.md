# 개발 내용 정리 (상세)

## 1. 프로젝트 개요
- **로컬에서 동작하는 옵시디언 스타일의 Markdown 에디터**
- Express 서버가 파일 시스템의 `notes/` 폴더를 읽고/저장하는 구조
- 프론트엔드는 단일 페이지(HTML/CSS/JS)로 구성, `marked`로 Markdown 미리보기 렌더링
- 로컬 스토리지에 즐겨찾기, 최근 문서, 태그, 색상, 버전 히스토리, 템플릿 저장

---

## 2. 백엔드(서버) 주요 기능 (`server.js`)

### 2.1 문서/폴더 CRUD
- **문서 목록 조회**: `GET /api/notes`
  - `notes/` 내부의 `.md` 파일을 재귀적으로 수집
- **폴더 목록 조회**: `GET /api/folders`
  - `notes/` 내부 폴더 구조를 재귀적으로 수집
- **문서 로드**: `GET /api/notes/:name`
- **문서 저장/생성**: `POST /api/notes`
- **문서 삭제**: `DELETE /api/notes/:name`
- **문서 이름 변경(이동 포함)**: `PUT /api/notes/:name/rename`
- **문서 복사**: `POST /api/notes/:name/copy`
- **폴더 생성**: `POST /api/folders`
- **폴더 삭제(재귀)**: `DELETE /api/folders/*`
- **폴더 이름 변경/이동**: `PUT /api/folders/rename`

### 2.2 검색
- **문서 이름/내용 검색**: `GET /api/search?q=...`
  - 모든 문서를 읽어 단순 포함 검색

### 2.3 보안/정규화
- 경로 정규화 및 금지 문자 필터로 **디렉터리 탈출 방지**
- `.normalize("NFC")` 기반 유니코드 정규화

### 2.4 AI 기능
- **AI 질문**: `POST /api/ai/ask`
  - 입력: `content`, `question`
  - 시스템 프롬프트:
    - `당신은 제공된 문서를 기반으로 질문에 답하는 도우미 입니다. 답변은 간결한 Markdown으로 작성하세요.`
  - 모델: `OPENAI_MODEL` 환경변수 (기본 `gpt-4o-mini`)
- **자동 태그 생성**: `POST /api/tags/auto`
  - 입력: `content`
  - 출력: 태그 최대 5개
  - 태그 파싱/정제 로직 포함

### 2.5 LangSmith 로깅
- `LANGSMITH_API_KEY` 설정 시 `ai-ask`, `auto-tags` 실행 로그 기록

## 3. 프론트엔드 주요 기능 (`public/*`)

### 3.1 문서 편집/미리보기
- **Markdown 편집기 + 미리보기** (Split / Edit / Preview 모드)
- `marked`가 없을 경우 **기본 Markdown 렌더링 fallback**
- **작업 체크박스 토글 지원** (미리보기에서 클릭 시 원문 업데이트)
- **탭 키 들여쓰기 지원**

### 3.2 자동 저장 & 상태 표시
- 일정 시간(800ms) 디바운스 기반 **자동 저장**
- 저장 상태 표시: 저장됨 / 변경 있음 / 저장 중

### 3.3 검색 기능
- **문서 목록 검색** (서버 `/api/search` 호출)
- **문서 내 검색**
  - 결과 하이라이트, 이전/다음 이동
  - 에디터/미리보기 동기 스크롤

### 3.4 폴더 트리 & 드래그앤드롭
- 폴더 트리 UI 렌더링
- **노트를 드래그하여 폴더 이동**
- **파일 드래그 드롭으로 가져오기** (txt, md 등 허용 확장자)

### 3.5 즐겨찾기/최근 문서
- 즐겨찾기/최근 문서 리스트 관리 (localStorage)

### 3.6 태그 관리
- 문서별 태그 저장 및 렌더링
- 사이드바 태그 필터
- 태그 칩 UI 및 개별 제거
- **자동 태그 생성** 버튼

### 3.7 색상 커스터마이징
- 문서/폴더 색상 지정 (컬러 피커 UI)

### 3.8 버전 히스토리
- 문서 저장 시 **버전 스냅샷 저장 (최대 20개)**
- 히스토리 패널에서 복원 가능

### 3.9 템플릿 기능
- 기본 템플릿(회의록, 일일 기록, 아이디어)
- 사용자 커스텀 템플릿 저장/삽입

### 3.10 UI/UX 개선
- 상단 툴바/태그 패널이 고정되어 보이도록 스크롤 구조 조정
- 토스트 메시지 기반 사용자 피드백
- 단축키 지원
  - 저장(Ctrl/Cmd+S), 새 문서(Ctrl/Cmd+N), 검색(Ctrl/Cmd+F), 보기 전환(1/2/3), 히스토리(Shift+H)
