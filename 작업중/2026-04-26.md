3. 헤딩도 없으면 단일 슬라이드

## 3. AI 요약 + 그래픽화 + 발표 스크립트 (2차)

### 서버 변경
- 신규 엔드포인트 **`POST /api/ai/pptify`**
- OpenAI `gpt-4o-mini` + `response_format: json_object` (강건한 JSON)
- Langsmith 트래킹 통합 (`name: "ai-pptify"`)

### 4종 시각 레이아웃 (AI가 자동 선택)
| 레이아웃 | 용도 | PPTX 표현 |
|---|---|---|
| `bullets` | 일반 핵심 요점 | 글머리 기호 리스트 |
| `comparison` | A vs B, 장단점 비교 | 좌우 2열 컬러 카드 |
| `process` | 순서/단계/흐름 | 번호 박스 + 화살표 |
| `stat` | 핵심 숫자/지표 강조 | 대형 숫자 + 캡션 |

### 발표 스크립트
- 각 슬라이드마다 **2분 분량** (한국어 500~700자) 자동 생성
- 도입 → 핵심 설명 → 예시 → 다음 슬라이드 전환 흐름
- 미리보기에 🎤 아이콘 + 글자 수 + 예상 발화 시간 표시
- PPTX `addNotes()`로 **발표자 노트에 자동 첨부** → PowerPoint 발표자 보기에서 그대로 확인

### UI 추가
- 모달 푸터: **"🤖 AI 요약 + 그래픽화"** + **"↺ 원본"** 버튼
- 레이아웃별 색상 배지 (bullets=파랑, comparison=노랑, process=초록, stat=핑크)
- 로딩 상태 표시 (`AI 분석 중...` / `✓ 완료` / `✕ 실패`)

## 미해결 사항

- **서버 재시작 필요**: 기존 Node 프로세스(PID 17820)에 새 `/api/ai/pptify` 엔드포인트가 반영되지 않음
  - 사용자가 직접 재시작하거나 권한 부여 시 PID 종료 후 재기동 가능
- **브라우저 E2E 검증 필요**: 모달 레이아웃, AI 요약 결과 품질, PPTX 다운로드 후 PowerPoint 호환성

## 검증 방법

1. 기존 서버 종료 후 `npm start` 재실행
2. `http://localhost:3001` → `Ctrl+F5`로 캐시 갱신
3. 마크다운 작성 후 "📊 PPT" 클릭 → 모달 열림 확인
4. "🤖 AI 요약 + 그래픽화" 클릭 → AI 응답 후 4종 레이아웃 자동 선택 확인
5. 각 슬라이드 미리보기에 발표 스크립트 박스 표시 확인
6. "다운로드 PPTX" → PowerPoint에서 열어 슬라이드 + 발표자 노트 확인
